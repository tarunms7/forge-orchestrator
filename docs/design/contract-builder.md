## 1. Problem Statement

### What Breaks Today

When Forge splits a feature into backend (BE) and frontend (FE) tasks, the agents
build code in **isolated git worktrees**. Neither agent can see the other's code
during development. This leads to **API contract mismatches** — the most critical
class of bugs in Forge pipelines.

### Real Example (Feature 3B — Pipeline Templates)

The planner created 6 tasks. Task-3 built the backend templates API, Task-5 built
the `TemplatePicker` component, Task-6 built the Settings page.

**What task-3 (backend) built:**
```python
# GET /api/templates returns:
TemplateListResponse(
    builtin=[...],   # list of PipelineTemplate
    user=[...],      # list of PipelineTemplate
)
# Each PipelineTemplate has fields: id, name, description, icon,
# model_strategy, build_cmd, test_cmd, review_config, ...
```

**What task-5 (frontend) assumed:**
```typescript
// Expected a FLAT ARRAY from GET /api/templates:
const templates = await fetch("/api/templates");
const data = await response.json() as PipelineTemplate[];
// Used wrong field names: build_command, test_command, skip_llm_review
```

**Result:** 5 bugs (2 critical, 1 high, 2 medium). The UI was completely non-functional
for user templates. Every CRUD operation was broken because:
1. Response shape mismatch (`{ builtin, user }` vs flat array)
2. Field name mismatch (`build_cmd` vs `build_command`, `skip_l2` vs `skip_llm_review`)
3. ID handling mismatch (backend returns UUID, frontend fabricated IDs)

### Why Current Solutions Don't Help

| Solution | Why It Fails |
|----------|-------------|
| **Sibling context** (PR #52) | Tells reviewer "task-3 owns templates.py" but NOT what the API returns |
| **Dependency context** | Tells agent "task-3 built pipeline templates, files: routes.py" — a summary, not a schema |
| **Making FE depend on BE** | Agent CAN read BE code but doesn't THINK to. Also: forces sequential execution, wastes parallelism |
| **Per-task LLM review** | Reviewer sees one task's diff in isolation — can't detect cross-task mismatches |

## 2. Root Cause Analysis

The fundamental issue is that the task graph captures **file-level dependencies**
but not **API-contract-level dependencies**.

```
WHAT THE DAG MODELS:
  task-3 owns: forge/api/routes/templates.py, forge/core/templates.py
  task-5 owns: web/src/components/task/TemplatePicker.tsx
  task-5 depends_on: [task-3]  (ordering only)

WHAT THE DAG MISSES:
  task-5 CONSUMES the API contract defined by task-3:
    GET /api/templates → { builtin: Template[], user: Template[] }
    POST /api/templates → CreateTemplateRequest → TemplateResponse
    Template = { id, name, description, icon, model_strategy, ... }
```

The planner creates tasks with clear descriptions ("Build the templates REST API",
"Build the TemplatePicker component") but never explicitly states the interface
between them. Each agent independently invents what it thinks the interface looks like.

**This is exactly how humans break things too** — which is why real engineering teams
define API specs (OpenAPI, Protobuf, GraphQL schemas) before coding starts. Both
teams build against the spec, not against each other's assumptions.

---

## 3. Design Goals

### Primary Goal
**Eliminate API contract mismatches between parallel tasks** by defining precise
interface contracts before any agent starts coding.

### Secondary Goals
1. **Maximize parallelism** — contracts make `depends_on` unnecessary for BE/FE pairs.
   Both tasks get the same contract and build independently.
2. **Scale to massive projects** — contracts are small, focused context. Not full code dumps.
3. **Zero false positives** — contracts are generated by an LLM that reads the plan and
   existing code. Not hallucinated from thin air.
4. **Backward compatible** — pipelines without cross-task interfaces skip the contract phase
   automatically (zero cost, zero latency).

### Non-Goals (for V1)
- Phase-based execution (designed here, implemented later)
- Automatic OpenAPI spec generation from existing code
- Contract version management across pipeline runs

## 4. Architecture Overview

### Current Pipeline Flow
```
User Request
    ↓
  PLANNER (1 LLM call)
    ↓
  TaskGraph { tasks, conventions }
    ↓
  EXECUTE (parallel agents)
    ↓
  REVIEW (per-task, 3-gate)
    ↓
  MERGE → PR
```

### New Pipeline Flow
```
User Request
    ↓
  PLANNER (1 LLM call)  ← enhanced: outputs integration_hints per task
    ↓
  TaskGraph { tasks, conventions }
    ↓
  CONTRACT BUILDER (1 LLM call)  ← NEW PHASE
    ↓
  ContractSet { api_contracts, type_contracts }
    ↓
  EXECUTE (parallel agents)  ← each agent gets relevant contracts
    ↓
  REVIEW (per-task, 3-gate)  ← reviewer verifies contract compliance
    ↓
  MERGE → PR
```

### Key Design Decisions

1. **Separate LLM call** (not part of planner): Single responsibility. Planner decomposes
   tasks. Contract Builder defines interfaces. Different concerns, different prompts,
   independently retriable.

2. **Planner hints → Contract Builder refines**: The planner flags integration points
   ("task-5 consumes API from task-3"). The Contract Builder reads those hints +
   existing codebase to generate precise schemas.

3. **Contracts in agent prompts**: Each agent receives ONLY the contracts relevant to
   its task — as a "producer" (you must build this exact API) or "consumer" (you must
   call this exact API shape).

4. **Contract verification at review**: The L2 reviewer gets contracts as additional
   review criteria: "Verify the implementation matches the contract."

5. **Auto-skip when no contracts needed**: If the planner produces no integration hints
   (e.g., all tasks are independent), the contract phase is skipped entirely.

## 5. Data Models

### New File: `forge/core/contracts.py`

```python
"""Contract models for cross-task interface alignment."""

from __future__ import annotations
from enum import Enum
from pydantic import BaseModel, Field


class ContractType(str, Enum):
    """Types of cross-task contracts."""
    API_ENDPOINT = "api_endpoint"
    SHARED_TYPE = "shared_type"
    EVENT = "event"
    FILE_IMPORT = "file_import"


class FieldSpec(BaseModel):
    """A single field in a type contract."""
    name: str
    type: str              # e.g., "string", "number", "boolean", "Template[]"
    required: bool = True
    description: str = ""


class TypeContract(BaseModel):
    """Contract for a shared data structure used across tasks."""
    name: str                   # e.g., "PipelineTemplate", "ReviewConfig"
    description: str = ""
    fields: list[FieldSpec]
    used_by_tasks: list[str]    # task IDs that reference this type


class APIContract(BaseModel):
    """Contract for a single API endpoint."""
    id: str                     # e.g., "contract-api-1"
    method: str                 # GET, POST, PUT, DELETE, PATCH
    path: str                   # e.g., "/api/templates"
    description: str = ""
    request_body: list[FieldSpec] | None = None
    response_body: list[FieldSpec]
    response_example: str = ""  # JSON example string for clarity
    auth_required: bool = True
    producer_task_id: str       # task that implements this endpoint
    consumer_task_ids: list[str]  # tasks that call this endpoint


class IntegrationHint(BaseModel):
    """A cross-task interface flagged by the planner.

    The planner doesn't define contracts — it just flags where they exist.
    The Contract Builder uses these hints to generate precise contracts.
    """
    producer_task_id: str       # task that creates/defines the interface
    consumer_task_ids: list[str]  # tasks that consume the interface
    interface_type: ContractType
    description: str            # e.g., "REST API for template CRUD"
    # Optional hints from planner to guide contract generation
    endpoint_hints: list[str] = Field(default_factory=list)
        # e.g., ["GET /api/templates", "POST /api/templates"]


class ContractSet(BaseModel):
    """All contracts for a pipeline. Output of the Contract Builder."""
    api_contracts: list[APIContract] = Field(default_factory=list)
    type_contracts: list[TypeContract] = Field(default_factory=list)
    # Original hints (kept for traceability)
    integration_hints: list[IntegrationHint] = Field(default_factory=list)

    def contracts_for_task(self, task_id: str) -> TaskContracts:
        """Get only the contracts relevant to a specific task."""
        producing_apis = [c for c in self.api_contracts if c.producer_task_id == task_id]
        consuming_apis = [c for c in self.api_contracts if task_id in c.consumer_task_ids]
        relevant_types = [
            t for t in self.type_contracts if task_id in t.used_by_tasks
        ]
        return TaskContracts(
            producing=producing_apis,
            consuming=consuming_apis,
            types=relevant_types,
        )

    def has_contracts(self) -> bool:
        """Whether any contracts exist (used to decide if phase should run)."""
        return bool(self.api_contracts or self.type_contracts)


class TaskContracts(BaseModel):
    """Contracts relevant to a single task. Injected into agent prompt."""
    producing: list[APIContract] = Field(default_factory=list)
    consuming: list[APIContract] = Field(default_factory=list)
    types: list[TypeContract] = Field(default_factory=list)

    def format_for_agent(self) -> str:
        """Format contracts as a system prompt section for the agent."""
        if not self.producing and not self.consuming and not self.types:
            return ""

        parts: list[str] = [
            "## Interface Contracts (STRICT — you MUST implement these exactly)",
            "",
            "Other tasks in this pipeline depend on these interfaces being exact.",
            "Do NOT deviate from the specified field names, types, or response shapes.",
            "",
        ]

        if self.types:
            parts.append("### Shared Types")
            parts.append("")
            for t in self.types:
                parts.append(f"**{t.name}**: {t.description}")
                parts.append("```")
                for f in t.fields:
                    req = "" if f.required else " (optional)"
                    parts.append(f"  {f.name}: {f.type}{req}  // {f.description}" if f.description else f"  {f.name}: {f.type}{req}")
                parts.append("```")
                parts.append("")

        if self.producing:
            parts.append("### APIs You Are PRODUCING (other tasks depend on these exact shapes)")
            parts.append("")
            for api in self.producing:
                parts.append(f"**{api.method} {api.path}** — {api.description}")
                if api.request_body:
                    parts.append("Request body:")
                    parts.append("```")
                    for f in api.request_body:
                        req = "" if f.required else " (optional)"
                        parts.append(f"  {f.name}: {f.type}{req}")
                    parts.append("```")
                parts.append("Response:")
                parts.append("```")
                for f in api.response_body:
                    req = "" if f.required else " (optional)"
                    parts.append(f"  {f.name}: {f.type}{req}")
                parts.append("```")
                if api.response_example:
                    parts.append(f"Example: `{api.response_example}`")
                parts.append("")

        if self.consuming:
            parts.append("### APIs You Are CONSUMING (use these exact shapes in your code)")
            parts.append("")
            for api in self.consuming:
                parts.append(f"**{api.method} {api.path}** — {api.description}")
                if api.request_body:
                    parts.append("Request body:")
                    parts.append("```")
                    for f in api.request_body:
                        req = "" if f.required else " (optional)"
                        parts.append(f"  {f.name}: {f.type}{req}")
                    parts.append("```")
                parts.append("Response shape:")
                parts.append("```")
                for f in api.response_body:
                    req = "" if f.required else " (optional)"
                    parts.append(f"  {f.name}: {f.type}{req}")
                parts.append("```")
                if api.response_example:
                    parts.append(f"Example: `{api.response_example}`")
                parts.append("")

        return "\n".join(parts)

    def format_for_reviewer(self) -> str:
        """Format contracts as review criteria for the L2 reviewer."""
        if not self.producing and not self.consuming:
            return ""

        parts: list[str] = [
            "## Contract Compliance Check",
            "",
            "This task has interface contracts with other tasks. "
            "VERIFY that the implementation matches these contracts EXACTLY.",
            "",
        ]

        for api in self.producing:
            parts.append(f"- **{api.method} {api.path}**: Must return EXACTLY these fields: "
                         + ", ".join(f"`{f.name}` ({f.type})" for f in api.response_body))

        for api in self.consuming:
            parts.append(f"- Code calling **{api.method} {api.path}** must expect EXACTLY these fields: "
                         + ", ".join(f"`{f.name}` ({f.type})" for f in api.response_body))

        parts.append("")
        parts.append("If any field names, types, or response shapes don't match the contract, FAIL the review.")

        return "\n".join(parts)
```

class TaskDefinition(BaseModel):
    """A task as defined by the planner. Immutable spec."""

    id: str = Field(min_length=1)
    title: str = Field(min_length=1)
    description: str
    files: list[str]
    depends_on: list[str] = Field(default_factory=list)
    complexity: Complexity = Complexity.MEDIUM
    # NEW: Integration hints from planner
    integration_hints: list[dict] | None = None
        # Each dict: { producer_task_id, consumer_task_ids, interface_type,
        #              description, endpoint_hints }
        # None = no cross-task interfaces for this task

    @field_validator("files")
    @classmethod
    def files_not_empty(cls, v: list[str]) -> list[str]:
        if not v:
            raise ValueError("Task must declare at least one file")
        return v

## 6. Enhanced Planner Output

### Changes to `PLANNER_SYSTEM_PROMPT` in `forge/core/claude_planner.py`

Add the following to the schema example and rules:

```
{
  "conventions": { ... },
  "tasks": [ ... ],
  "integration_hints": [
    {
      "producer_task_id": "task-1",
      "consumer_task_ids": ["task-3", "task-4"],
      "interface_type": "api_endpoint",
      "description": "REST API for template CRUD",
      "endpoint_hints": ["GET /api/templates", "POST /api/templates", "PUT /api/templates/{id}", "DELETE /api/templates/{id}"]
    }
  ]
}
```

Add this to the rules section:

```
- INTEGRATION HINTS: When tasks have cross-task interfaces (one task produces an
  API/type/event that another task consumes), add an "integration_hints" array.
  Each hint identifies:
  - producer_task_id: The task that CREATES the interface (e.g., backend API)
  - consumer_task_ids: Tasks that CONSUME it (e.g., frontend components)
  - interface_type: one of "api_endpoint", "shared_type", "event", "file_import"
  - description: What the interface is for
  - endpoint_hints: (for api_endpoint only) List of endpoints like "GET /api/foo"
- If there are NO cross-task interfaces (all tasks are independent), omit
  integration_hints entirely.
- IMPORTANT: Integration hints enable PARALLEL execution. When you add integration
  hints, the system can generate contracts so both producer and consumer tasks
  run simultaneously. Without hints, consumer tasks must wait for producer tasks.
  PREFER adding hints over adding depends_on for API integration tasks.
```

## 7. Contract Builder LLM

### New File: `forge/core/contract_builder.py`

The Contract Builder is a focused LLM call that reads the TaskGraph + integration hints
and generates precise interface contracts.

```python
"""Contract Builder. Generates cross-task interface contracts from planner hints."""

import json
import logging
import re
from collections.abc import Callable

from claude_code_sdk import ClaudeCodeOptions

from forge.core.contracts import ContractSet, IntegrationHint
from forge.core.models import TaskGraph
from forge.core.sdk_helpers import SdkResult, sdk_query

logger = logging.getLogger("forge.contracts")


CONTRACT_BUILDER_SYSTEM_PROMPT = """You are an interface contract generator for a multi-agent coding system called Forge.

Given a task graph and integration hints, generate precise interface contracts that
multiple coding agents will build against simultaneously.

Your contracts must be EXACT — field names, types, and response shapes must be specific
enough that two independent developers could implement both sides and have them work
together on first try.

RULES:
- For each integration hint, generate one or more APIContract entries
- For shared data structures referenced by multiple contracts, generate TypeContract entries
- Use the existing codebase context to align with established patterns (e.g., if the
  project uses snake_case for API fields, use snake_case in contracts)
- If the hint references existing code (e.g., extending an existing API), READ the
  existing code first to ensure the contract is consistent
- response_example should be a realistic JSON string showing the response shape
- Each FieldSpec must have a clear type: use "string", "number", "boolean", "string[]",
  or reference a TypeContract name like "PipelineTemplate[]"
- Mark fields as required: true unless they are genuinely optional
- Include ALL fields in the response — don't omit fields the consumer will need

Output ONLY valid JSON matching this schema:

{
  "api_contracts": [
    {
      "id": "contract-api-1",
      "method": "GET",
      "path": "/api/templates",
      "description": "List all templates (built-in and user-created)",
      "request_body": null,
      "response_body": [
        { "name": "builtin", "type": "PipelineTemplate[]", "required": true, "description": "Built-in templates" },
        { "name": "user", "type": "PipelineTemplate[]", "required": true, "description": "User-created templates" }
      ],
      "response_example": "{\\"builtin\\": [{...}], \\"user\\": [{...}]}",
      "auth_required": true,
      "producer_task_id": "task-1",
      "consumer_task_ids": ["task-2"]
    }
  ],
  "type_contracts": [
    {
      "name": "PipelineTemplate",
      "description": "A pipeline configuration template",
      "fields": [
        { "name": "id", "type": "string", "required": true, "description": "Unique identifier (UUID for user, slug for built-in)" },
        { "name": "name", "type": "string", "required": true, "description": "" }
      ],
      "used_by_tasks": ["task-1", "task-2"]
    }
  ]
}

JSON only. No markdown fences, no explanation."""


class ContractBuilderLLM:
    """Generates interface contracts from planner integration hints."""

    def __init__(
        self,
        model: str = "sonnet",
        cwd: str | None = None,
    ) -> None:
        self._model = model
        self._cwd = cwd
        self._last_sdk_result: SdkResult | None = None

    async def generate_contracts(
        self,
        graph: TaskGraph,
        hints: list[IntegrationHint],
        project_context: str = "",
        on_message: Callable | None = None,
    ) -> str:
        """Generate a ContractSet JSON string from integration hints."""
        prompt = self._build_prompt(graph, hints, project_context)

        options = ClaudeCodeOptions(
            system_prompt=CONTRACT_BUILDER_SYSTEM_PROMPT,
            max_turns=10,  # Allow reading existing code for context
            model=self._model,
            allowed_tools=["Read", "Glob", "Grep", "Bash"],
            permission_mode="acceptEdits",
        )
        if self._cwd:
            options.cwd = self._cwd

        try:
            result = await sdk_query(
                prompt=prompt, options=options, on_message=on_message,
            )
        except Exception as e:
            logger.warning("SDK call failed during contract generation: %s", e)
            return ""

        self._last_sdk_result = result
        result_text = result.result if result and result.result else ""
        return _extract_json(result_text)

    def _build_prompt(
        self,
        graph: TaskGraph,
        hints: list[IntegrationHint],
        project_context: str,
    ) -> str:
        parts: list[str] = []

        # Task summaries so the builder understands what each task does
        parts.append("## Task Graph\n")
        for task in graph.tasks:
            parts.append(
                f"- **{task.id}** ({task.title}): {task.description}\n"
                f"  Files: {', '.join(task.files)}"
            )

        parts.append("\n## Integration Hints\n")
        for hint in hints:
            parts.append(
                f"- Producer: {hint.producer_task_id} → "
                f"Consumers: {', '.join(hint.consumer_task_ids)}\n"
                f"  Type: {hint.interface_type}\n"
                f"  Description: {hint.description}"
            )
            if hint.endpoint_hints:
                parts.append(
                    f"  Endpoints: {', '.join(hint.endpoint_hints)}"
                )

        if project_context:
            parts.append(f"\n## Project Context\n{project_context}")

        parts.append(
            "\nGenerate precise contracts for ALL integration hints above. "
            "Read existing code if needed to align with project patterns. "
            "Respond with ONLY the ContractSet JSON."
        )

        return "\n\n".join(parts)


class ContractBuilder:
    """Orchestrates contract generation with validation and retry."""

    def __init__(self, llm: ContractBuilderLLM, max_retries: int = 3) -> None:
        self._llm = llm
        self._max_retries = max_retries

    async def build(
        self,
        graph: TaskGraph,
        hints: list[IntegrationHint],
        project_context: str = "",
        on_message: Callable | None = None,
    ) -> ContractSet:
        """Generate and validate a ContractSet."""
        feedback: str | None = None

        for attempt in range(self._max_retries):
            logger.info("Contract generation attempt %d/%d", attempt + 1, self._max_retries)
            raw = await self._llm.generate_contracts(
                graph, hints, project_context, on_message,
            )
            contract_set, error = self._parse_and_validate(raw, graph)
            if contract_set is not None:
                logger.info("Contract generation succeeded on attempt %d", attempt + 1)
                return contract_set
            logger.warning("Attempt %d validation failed: %s", attempt + 1, error)
            # Future: feed error back to LLM for retry

        # If all retries fail, return empty ContractSet (graceful degradation)
        logger.warning("Contract generation failed after %d retries — proceeding without contracts", self._max_retries)
        return ContractSet()

    def _parse_and_validate(
        self, raw: str, graph: TaskGraph,
    ) -> tuple[ContractSet | None, str | None]:
        """Parse JSON and validate contract references."""
        if not raw or not raw.strip():
            return None, "Empty response"

        try:
            data = json.loads(raw)
        except json.JSONDecodeError as e:
            return None, f"Invalid JSON: {e}"

        try:
            contract_set = ContractSet.model_validate(data)
        except Exception as e:
            return None, f"Schema validation failed: {e}"

        # Validate that all referenced task IDs exist in the graph
        task_ids = {t.id for t in graph.tasks}
        for api in contract_set.api_contracts:
            if api.producer_task_id not in task_ids:
                return None, f"API contract {api.id} references unknown producer task: {api.producer_task_id}"
            for consumer_id in api.consumer_task_ids:
                if consumer_id not in task_ids:
                    return None, f"API contract {api.id} references unknown consumer task: {consumer_id}"

        for type_contract in contract_set.type_contracts:
            for task_id in type_contract.used_by_tasks:
                if task_id not in task_ids:
                    return None, f"Type contract {type_contract.name} references unknown task: {task_id}"

        return contract_set, None


def _extract_json(text: str) -> str:
    """Extract JSON from response, stripping markdown fences if present."""
    text = text.strip()
    match = re.search(r"```(?:json)?\s*(\{.*?\})\s*```", text, re.DOTALL)
    if match:
        return match.group(1)
    start = text.find("{")
    end = text.rfind("}")
    if start != -1 and end != -1:
        return text[start : end + 1]
    return text
```

The ContractSet is a self-contained JSON blob. It's:
- Written once (after contract generation)
- Read many times (once per task dispatch)
- Never partially updated
- Small (typically < 10KB even for large pipelines)

No need for a separate table. The `_add_missing_columns()` auto-migration handles adding
this column to existing databases.

## 11. Pipeline Flow Changes

### Changes to `forge/core/daemon.py`

#### New Method: `generate_contracts()`

```python
async def generate_contracts(
    self,
    graph: TaskGraph,
    db: Database,
    pipeline_id: str,
) -> ContractSet:
    """Generate interface contracts from planner integration hints.

    Runs between plan() and execute(). Skips if no integration hints exist.
    """
    # Extract integration hints from the TaskGraph
    raw_hints = graph.integration_hints or []
    if not raw_hints:
        logger.info("No integration hints — skipping contract generation")
        return ContractSet()

    hints = [IntegrationHint.model_validate(h) for h in raw_hints]

    await self._emit(
        "pipeline:phase_changed",
        {"phase": "contracts"},
        db=db,
        pipeline_id=pipeline_id,
    )

    contract_model = select_model(self._strategy, "contract_builder", "high")
    builder_llm = ContractBuilderLLM(model=contract_model, cwd=self._project_dir)
    builder = ContractBuilder(builder_llm)

    async def _on_contract_msg(msg):
        text = _extract_text(msg)
        if text:
            await self._emit(
                "contracts:output",
                {"line": text},
                db=db,
                pipeline_id=pipeline_id,
            )

    context = self._snapshot.format_for_planner() if self._snapshot else ""
    contract_set = await builder.build(
        graph, hints, project_context=context, on_message=_on_contract_msg,
    )

    # Track cost
    if builder_llm._last_sdk_result:
        sdk_result = builder_llm._last_sdk_result
        if sdk_result.cost_usd > 0:
            await db.add_pipeline_cost(pipeline_id, sdk_result.cost_usd)

    # Persist contracts
    if contract_set.has_contracts():
        await db.set_pipeline_contracts(
            pipeline_id, contract_set.model_dump_json(),
        )
        console.print(
            f"[green]Contracts: {len(contract_set.api_contracts)} API, "
            f"{len(contract_set.type_contracts)} types[/green]"
        )
    else:
        console.print("[yellow]Contract generation produced no contracts[/yellow]")

    await self._emit(
        "pipeline:contracts_ready",
        {
            "api_count": len(contract_set.api_contracts),
            "type_count": len(contract_set.type_contracts),
        },
        db=db,
        pipeline_id=pipeline_id,
    )

    return contract_set
```

#### Updated `run()` Method

```python
async def run(self, user_input: str) -> None:
    """Full pipeline for CLI: plan → contracts → execute."""
    db_path = os.path.join(self._project_dir, ".forge", "forge.db")
    os.makedirs(os.path.dirname(db_path), exist_ok=True)
    db = Database(f"sqlite+aiosqlite:///{db_path}")
    await db.initialize()
    try:
        self._pipeline_id = str(uuid.uuid4())
        await db.create_pipeline(
            id=self._pipeline_id, description=user_input[:200],
            project_dir=self._project_dir, model_strategy=self._strategy,
            budget_limit_usd=self._settings.budget_limit_usd,
        )
        graph = await self.plan(user_input, db, pipeline_id=self._pipeline_id)

        # NEW: Contract generation phase
        self._contracts = await self.generate_contracts(graph, db, self._pipeline_id)

        try:
            await check_budget(db, self._pipeline_id, self._settings)
        except BudgetExceededError as exc:
            await self._emit("pipeline:budget_exceeded", {
                "spent": exc.spent,
                "limit": exc.limit,
            }, db=db, pipeline_id=self._pipeline_id)
            await db.update_pipeline_status(self._pipeline_id, "error")
            raise
        await self.execute(graph, db, pipeline_id=self._pipeline_id)
    finally:
        await db.close()

## 13. Future: Phase-Based Execution

> **This section describes V2 architecture. Not implemented in V1 but designed here
> for forward compatibility.**

### The Vision: 1000 Parallel Claudes

For massive projects, the planner needs to break work into phases. But phases shouldn't
be strictly sequential — if Phase 2's contracts are defined upfront, Phase 2 tasks can
start as soon as their contracts are ready, even while Phase 1 is still running.

### Phase Model

```python
class Phase(BaseModel):
    """A group of tasks that can execute in parallel."""
    id: str                  # e.g., "phase-1"
    title: str
    tasks: list[TaskDefinition]
    depends_on_phases: list[str] = Field(default_factory=list)
    # Contracts for this phase's tasks — defined BEFORE phase executes
    contracts: ContractSet | None = None


class PhasedTaskGraph(BaseModel):
    """Extended TaskGraph with phase groupings."""
    phases: list[Phase]
    conventions: dict | None = None
    # Cross-phase contracts (defined by Contract Builder for ALL phases)
    global_contracts: ContractSet | None = None
```

### How Phases Overlap

```
Phase 1: [task-1, task-2, task-3]  ← all run in parallel
Phase 2: [task-4, task-5, task-6]  ← all run in parallel

TRADITIONAL (sequential):
  Phase 1 completes → Phase 2 starts
  Total time: Phase 1 time + Phase 2 time

WITH CONTRACTS (overlapping):
  Contract Builder defines contracts for BOTH phases upfront
  Phase 1 starts → Phase 2 starts 50% through Phase 1
  Phase 2 tasks build against contracts (not Phase 1's code)
  Phase 1 code must match the same contracts
  Total time: max(Phase 1, Phase 2) + overlap buffer
```

### Hierarchical Planning (V3)

For truly massive projects (100+ tasks), even the Contract Builder can't run in one
call. The system becomes hierarchical:

```
Meta-Planner (breaks project into phases)
    ↓
Phase Planners (one per phase, run in parallel)
    ↓
Contract Builder (one per phase pair with cross-phase interfaces)
    ↓
Agents (all run in parallel within and across phases)
```

### Forward Compatibility in V1

The V1 data models are designed so that phased execution can be added later:
- `ContractSet` is a standalone JSON blob — can be extended with phase references
- `IntegrationHint` can be extended with `phase_id` fields
- `PipelineRow.contracts_json` stores any ContractSet structure
- The `contracts_for_task()` method already filters by task ID, regardless of phases

## Appendix A: How This Would Have Prevented the 3B Bugs

### Bug 1 (Critical): Response shape mismatch

**Contract would have said:**
```

```

**Frontend agent would have seen:**
```
### APIs You Are CONSUMING
GET /api/templates — Response shape:
  builtin: PipelineTemplate[]
  user: PipelineTemplate[]
```

The agent would have written `response.user` instead of `response as PipelineTemplate[]`.

### Bug 2 (Critical): Field name mismatch

**Contract would have defined the shared type:**
```
PipelineTemplate:
  id: string
  name: string
  build_cmd: string        ← NOT "build_command"
  test_cmd: string         ← NOT "test_command"
  review_config: ReviewConfig  ← NOT "skip_llm_review" as a top-level boolean
```

The frontend agent would have used the correct field names from the contract.

### All 5 Bugs

Every bug from 3B was an API contract mismatch. With contracts, **zero** of them would
have existed. Both agents would have built against the same spec, independently, in parallel.

## Appendix C: The Answer to "Why Not Just Make FE Depend on BE?"

Making FE depend on BE means:

```
Timeline (sequential):
  [===== BE task (3 min) =====]→[===== FE task (3 min) =====]
  Total: 6 minutes

Timeline (with contracts, parallel):
  [===== BE task (3 min) =====]
  [===== FE task (3 min) =====]
  Total: 3 minutes
```

For a pipeline with 10 BE/FE pairs:
- Sequential: 60 minutes
- Parallel with contracts: 6 minutes (10x speedup)

For 1000 parallel Claude sessions, this difference is existential. Contracts are the
mechanism that transforms Forge from a sequential pipeline into a massively parallel
build system.
